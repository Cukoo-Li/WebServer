# 池资源

## 线程池

线程池中有特定数量的分离线程，它们不断循环阻塞等待任务资源，一旦被唤醒，就尝试从任务队列中取出任务并处理之。

- 数据成员
  - 描述线程池的数组（线程数量固定）
  - 任务队列（限制最大任务数）
  - 锁（用于保护任务队列）
  - 信号量（用于描述任务资源数目）
  - 标志量（是否结束线程，在线程执行的函数中作为循环条件）
- 成员函数
  - 构造函数（创建线程，并设置为脱离）
  - 析构函数（将标志量设置为结束）
  - `append`（往任务队列中添加任务）
  - `run`（线程执行的函数，循环阻塞等待信号量，被唤醒时尝试取出一个任务并处理之）
  - `worker`（静态函数，用于传给`pthread_create`）

## 数据库连接池

将多个数据库连接放入一个队列中，供工作线程取用，工作线程用完后需要归还连接。用RAII手法封装取用连接和归还连接的操作。

- 数据成员
  - 存放连接的队列
  - 连接数量
  - 锁（用于保护连接队列）
  - 信号量（用于描述连接资源数目）
  - 与数据库连接创建有关的变量（主机名、用户名、密码、数据库名、数据库端口）
- 成员函数
  - 初始化数据库连接池（需要给定主机名、用户名、密码、数据库名、数据库端口）
  - 获取连接
  - 归还连接
